New features / improvements

Version 3.98

  1. GUI Core:
    • New function GUI_DispStringInRectWrap() added. Draws text with optional text wrapping within the given rectangle.
    • New function GUI_GIF_DrawEx() added. Draws the given sub item of a GIF file.
    • New function GUI_GIF_GetInfo() added. Returns information about a GIF file.
    • New function GUI_JPEG_DrawEx() added. Draws a JPEG file which does not have to be loaded completely into memory.
    • New function GUI_JPEG_DrawScaled() added. Draws a JPEG file with optional scaling.
    • New function GUI_JPEG_DrawScaledEx() added. Draws a JPEG file which does not have to be loaded completely into memory with optional scaling.
    • New function GUI_JPEG_GetInfoEx() added. Returns information about a JPEG file which does not have to be loaded completely into memory.
    • New function GUI_CURSOR_GetState() added. Returns if the cursor is visible or not.
  2. Memory devices:
    • New function GUI_MEMDEV_MarkDirty() added. Marks pixels as 'used' which have been modified by a pointer operation.
  3. Widget library:
    • New config option WIDGET_USE_PARENT_EFFECT added. If set to 1, each child widget has the same effect as its parent widget.
    • The key GUI_KEY_BACKTAB now can be used to move to the previous element of a dialog item. In the simulation SHIFT+TAB can be used.
  4. EDIT widget:
    • New function EDIT_GetCursorCharPos() added. Returns the character position of the cursor.
    • New function EDIT_GetCursorPixelPos() added. Returns the pixel position of the cursor.
    • New function EDIT_SetUlongMode() added. Sets the widget to unsigned long mode (0 - 4294967295).
    • EDIT widget now sends WM_NOTIFICATION_CLICKED and WM_NOTIFICATION_RELEASED messages.
  5. GRAPH widget:
    • New function GRAPH_DATA_XY_SetLineStyle() added. Sets the line style used to draw the data.
  6. HEADER widget:
    • New function HEADER_SetDragLimit() added. Limits dragging of the dividers within the window area of the header widget.
  7. LISTVIEW:
    • New function LISTVIEW_CompareDec() added. Compare function for comparing 2 integer values.
    • New function LISTVIEW_CompareText() added. Compare function for comparing 2 strings.
    • New function LISTVIEW_DisableSort() added. Disables sorting of the LISTVIEW widget.
    • New function LISTVIEW_EnableSort() added. Enables sorting of the LISTVIEW.
    • New function LISTVIEW_GetSelUnsorted() added. Returns the number of the selected item in unsorted state.
    • New function LISTVIEW_GetUserData() added. Returns the user data of the given row which has been set with the function LISTVIEW_SetUserData().
    • New function LISTVIEW_InsertRow() added. Inserts the given row at the given position.
    • New function LISTVIEW_SetAutoScrollH() added. Enables the automatic use of a horizontal scrollbar.
    • New function LISTVIEW_SetAutoScrollV() added. Enables the automatic use of a vertical scrollbar.
    • New function LISTVIEW_SetCompareFunc() added. Sets the compare function for the given column.
    • New function LISTVIEW_SetSelUnsorted() added. Sets the current selection in unsorted state.
    • New function LISTVIEW_SetSort() added. Sets the column and sorting order to be sorted by.
    • New function LISTVIEW_SetUserData() added. Sets the user data of the given row.
    • New config option LISTVIEW_SCROLLSTEP_H_DEFAULT added.
    • The keys GUI_KEY_RIGHT and GUI_KEY_LEFT can now be used for scrolling to the left and to the right.
  8. MENU widget:
    • New function MENU_Popup() added. Opens a popup menu at the given position.
  9. MULTIEDIT widget:
    • New function MULTIEDIT_GetCursorCharPos() added. Returns the character position of the cursor.
    • New function MULTIEDIT_GetCursorPixelPos() added. Returns the pixel position of the cursor.
    • New function MULTIEDIT_AddText() added. Adds additional text at the current cursor position.
  10. TEXT widget:
    • New function TEXT_SetWrapMode() added. Sets the wrap mode of a specified text widget.
    • New function TEXT_SetDefaultWrapMode() added. Sets the default wrap mode for new text widgets.
  11. MESSAGEBOX widget:
    • New config option GUI_MESSAGEBOX_CF_MOVEABLE added. Enables dragging of the box at the title bar.
  12. FRAMEWIN widget:
    • New config option FRAMEWIN_ALLOW_DRAG_ON_FRAME added. Enables dragging of the window at the window frame.
  13. SCROLLBAR widget:
    • New config option SCROLLBAR_THUMB_SIZE_MIN_DEFAULT added. Sets the minimum size of the widgets thumb area.
  14. Display drivers:
    • LCDSLin driver: Support for RAIO 8822/8803/8835 added.
    • LCD667XX driver: Support for Sharp LR38825 and Samsung S6D0117 added.
    • LCDPage1bpp driver: Support for UltraChip UC1601 added.

  8. RADIO widget:
    • New function RADIO_GetText() added. Returns the optional text of a RADIO widget.

  10. 2-D Graphic Library:
    • New function GUI_GetDrawMode() added. Returns the currently selected draw mode.
  11. Memory devices:
    • New function GUI_MEMDEV_WriteEx() added. Writes the contents of the given memory device into the currently selected memory device using alpha blending and scaling.
    • New function GUI_MEMDEV_WriteExAt() added. Writes the contents of the given memory device into the currently selected memory device at the specified position using alpha blending and scaling.
  12. Window manager:
    • New function WM_GetCallback() added. Returns a pointer to the callback function of the given window.
  13. Display drivers:
    • LCD667XX driver: Support for Samsung S6D0110A added.
    • LCD07X1 driver: Support for Sitronix ST7541 added.
    • LCD66750 driver: Support for Hitachi HD66753 added.

  16. Antialiasing:

  17. Unicode:

    • New function GUI_UC_ConvertUTF82UC() added. Converts a UTF-8 string into Unicode format.
  18. Display drivers:

    • New driver LCD13701 for Epson S1D13701 controller added.

Program corrections

Version 3.96f

  1. Display driver:
    • LCDPage1bpp driver always writes to display 0. Problem solved.

Version 3.96e

  1. BUTTON widget:
    • The function BUTTON_GetBitmap() does not return NULL if the requested bitmap is not available. Problem solved.

Version 3.96d

  1. BUTTON widget:
    • If a bitmap with index BUTTON_BI_DISABLED is used with the function BUTTON_SetBitmap(), the memory is not released when deleting the BUTTON widget. Problem solved.

Version 3.96c

  1. Widget library:
    • When working in a multitasking environment the creation process of some widgets can be interrupted by other tasks, which can lead to accessing uninitialized widgets. This affects only multitasking systems with more than one GUI task. Problem solved.

Version 3.96b

  1. LISTBOX widget:
    • When scrolling over a disabled item with GUI_KEY_UP or GUI_KEY_DOWN, the optional scrollbar is not updated. Problem solved.
  2. SLIDER widget:
    • The file SLIDER.c does not compile with the configuration option WM_SUPPORT_TRANSPARENCY = 0. Problem solved.
  3. HEADER widget:
    • After dragging the width of a column over the touch screen, the column width is changed when touching the screen somewhere else. The problem occurs only if mouse and touch screen support is configured simultaneously. Problem solved.
  4. Showing JPEGS's:
    • When drawing a JPEG image it appears one pixel below the desired position. Problem solved.
    • Drawing a JPEG image changes the current foreground color. Problem solved.
  5. Memory devices:
    • If the memory allocated for memory devices is > 32k on 16 bit systems sometimes the allocation of memory fails. Problem solved.

Version 3.96

  1. Showing GIF's:
    • In some cases the leftmost pixel of a GIF is not drawn. Problem solved.
  2. Multitasking environments:
    • Executing a dialog keeps the GUI locked and blocks it for other tasks. Problem solved.

Version 3.94f

  1. DROPDOWN widget:
    • When moving the input focus from an open DROPDOWN widget with GUI_KEY_TAB to the next dialog item, an invalid pointer was dereferenced. Problem solved.
    • When deleting a window with an open DROPDOWN widget, an invalid handle was dereferenced by the window manager and it may run into an endless loop. Problem solved.
  2. Window manager:
    • If memory devices were used for redrawing and insufficient memory was available to redraw the entire area, a memory defragmentation could occur which would lead to dereferencing of an invalid pointer. Problem solved.

Version 3.94e

  1. JPEG decompression:
    • Decompression of some larger JPEG images failed. Problem solved.

Version 3.94d

  1. LISTBOX widget:
    • When a LISTBOX was partially overlapped, it is sometimes not rendered properly. Problem solved.

Version 3.94c

  1. LISTVIEW widget:
    • When calling the function LISTVIEW_SetSel() with -1 as parameter NewSel the function crashes if the previous selection was >= 0. Problem solved.
  2. Memory devices:
    • If the memory allocated for memory devices is > 32k on 16 bit systems sometimes the allocation of memory fails. Problem solved.

Version 3.94b
